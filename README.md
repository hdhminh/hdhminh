## Introduction

I build software where AI is part of an operating system of work: document workflows, browser automation, human-in-the-loop tools, and traceable agent runtimes that need to behave predictably under real constraints.

My portfolio is centered on:

- LLM agents with runtime controls, memory, and traceability
- document and certification workflows for operator-managed environments
- browser automation and process tooling
- backend and frontend systems that turn AI into a usable product surface

## Featured systems

### Cyber LLM SOC Assistant

Production-oriented cybersecurity assistant built with FastAPI, Next.js, multi-agent reasoning, live trace monitoring, and benchmarked security analysis workflows.

My upstream contribution included:

- shared ReAct runtime helpers and runtime guardrails
- G1 and G2 execution flow improvements
- trace contract alignment between backend services and the web trace panel
- tests, benchmark artifacts, and supporting documentation

Repository:
- [cyber-llm-agent](https://github.com/hdhminh/cyber-llm-agent)

### PDF certification and web-signing workflow

Controlled PDF certification and browser-based signing workflow combining desktop control, local processing, Google Apps Script intake, and operator-managed document flow.

Highlights:

- desktop bridge plus browser signing flow
- local processing with controlled upload/return paths
- public-safe architecture and workflow documentation

Repository:
- [pdf-certification-workflow](https://github.com/hdhminh/pdf-certification-workflow)

### Dossier intake automation

Public-safe dossier intake automation built around Tkinter and Playwright, modeled on a real notarial operations workflow.

Highlights:

- browser-driven form handling
- operator workflow support
- documentation designed to preserve implementation privacy

Repository:
- [dossier-intake](https://github.com/hdhminh/dossier-intake)

### Medicinal herb assistant

AI assistant for Vietnamese medicinal herbs, combining a FastAPI backend, React frontend, Gemini-powered Q&A, and a computer-vision training pipeline.

My documented contribution focused on:

- the `CV_training` workspace
- training pipeline structure
- herb recognition experimentation support

Repository:
- [medicinal-herb-assistant](https://github.com/hdhminh/medicinal-herb-assistant)

### Mia Agent

Production-grade productivity agent showcasing LangGraph-based routing, pgvector semantic memory, and tool-driven execution design.

Repository:
- [mia-agent](https://github.com/hdhminh/mia-agent)
